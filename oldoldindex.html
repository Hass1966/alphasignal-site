The Honest Account

What the numbers
actually show.

A wealth manager once suggested I hand over my savings. He'd charge 5% a year — win or lose. That conversation sent me down a different path. Here is what I built, what worked, what didn't, and what it took to go from 36% accuracy to 56%.

// Where we started

The walk-forward backtest was humbling.

The first model generation — tested across 24 quarterly windows from Q1 2020 to Q4 2025 — produced a 36% BUY accuracy. We didn't hide it. We published it.

The problem wasn't model complexity. It was measurement honesty: the original metric included ambiguous HOLD-zone outcomes. Once we applied minimum return thresholds, the real picture emerged — and it was worse than we'd assumed. That kind of realisation is what makes a system better.

Crypto signals remain genuinely weak: 43–55% accuracy with free daily data. That is an honest ceiling, not a gap we expect to close soon.

// What actually moved the needle

Small, compounding improvements.

Each gain was incremental. They compound:

+2–3%
Sector momentum features. Adding 8 cross-sector correlation and momentum features gave models market context they were missing entirely.
+~1%
Platt scaling calibration. Raw model probabilities were overconfident. Calibration brought predicted probabilities in line with observed frequencies.
−15% false signals
Multi-horizon agreement filter. Requiring both 1-day and 5-day models to agree before signalling cut false positives substantially.
biggest gain
Fixing the measurement. The single largest improvement came from honest accuracy accounting — not from a fancier model.
Methodology

Walk-forward validation.
The way practitioners do it.

Traditional backtests train on the full dataset then measure on the same data — that is memorisation, not validation. Alpha Signal uses walk-forward validation exclusively: models see only what a real investor would have known at each point in time.

Yahoo Finance, Polygon free tier, CoinGecko, FRED macro data. 104 engineered features per asset — price, volume, volatility, momentum, macro indicators, and 8 sector momentum/correlation features. Zero paid data sources.

Q1 2020 through Q4 2025. Each window trains on all prior data, then tests on the next quarter with an expanding window. Models are retrained from scratch each fold. No future data ever enters the training set. 1,672 model files total.

The Ensemble

Six models.
One decision.

No single model maintains an edge across all market regimes. The ensemble combines classical statistics, gradient boosting, and deep learning — each casts a confidence-weighted vote. A BUY signal requires ensemble P(up) > 0.55, walk-forward accuracy > 51%, and both 1-day and 5-day horizon models in agreement.

// Classical · Gradient Descent
Linear Regression
Baseline return predictor. 3,000 epochs, 10 features. Captures linear factor exposures — momentum, mean-reversion, volume trends. Interpretable weights aid ensemble debugging.
Interpretable
// Classical · Sigmoid Loss
Logistic Regression
Binary UP/DOWN classifier. 3,000 epochs, regularised. Provides calibrated probability estimates via Platt scaling. Acts as the probabilistic anchor of the ensemble.
Probability-calibrated
// Boosting · Custom Rust CART
Gradient Boosted Trees
Custom Rust implementation — CART trees with boosting, no external library. Captures non-linear feature interactions. Robust to price outliers in trending markets.
Non-linear
// Boosting · C++ Bindings
LightGBM
Leaf-wise tree growth with histogram binning. Faster than XGBoost at comparable accuracy. Provides a second independent boosting vote, adding tree-based diversity.
High-speed
// Deep Learning · Candle-NN
LSTM + GRU
Long Short-Term Memory and Gated Recurrent Unit networks learn sequential price patterns across history. hidden_size=64, seq_length=10–20. The only model capturing temporal regime memory.

Platform Architecture

Four layers.
One NUC in London.

Built entirely in Rust, running 24/7 on a home Intel NUC11 (Ubuntu 24.04, 64 GB RAM). Every layer connects to the next via a typed REST API. No managed cloud services in the critical path. AutoHealer monitors all services and can restart any layer within seconds.

// Diagram 1 — Platform overview: all four layers
All four layers run as systemd services on a single Intel NUC11 (Ubuntu 24.04, 64 GB RAM). AutoHealer monitors all services and can restart any layer automatically. Open-source at github.com/Hass1966/autohealer
// Diagram 2 — Agent Alpha: autonomous observe-reason-act-reflect loop
The loop runs every hour. When accuracy drops below 45%, a retrain is proposed with a 48-hour cooldown to prevent thrashing. Regime detection adjusts all position sizes in real time — a CRISIS state halves maximum exposure within one cycle.

The 56% live accuracy comes from out-of-sample signals only — every prediction was made before the outcome was known. The system has run in this configuration since March 15, 2026.

56% doesn't sound dramatic. But with disciplined position sizing and a profit factor of 1.28, a consistent 6-point edge above random compounds meaningfully across hundreds of signals. The +12 bps expected value per signal is the number that matters operationally — it captures both win rate and average magnitude.

Stocks and FX are stronger performers at 57–65% accuracy in training windows. Crypto is the hardest domain with free daily data. The system distinguishes these and sizes accordingly.

The walk-forward backtest (Q1 2020 – Q4 2025, 24 windows) validated the methodology. The old model generation showed 36% accuracy. The current one, with sector features, Platt calibration, and the multi-horizon filter, reaches 56% live. Each improvement was small. They compound.
