The Honest Account

What the numbers
actually show.

Most trading system writeups cherry-pick the good numbers. This one doesn't. Here is what worked, what didn't, and what it took to go from 36% accuracy to 56% — and why that gap matters.

// Where we started

The walk-forward backtest was humbling.

The first generation of models — tested across 24 quarterly windows from Q1 2020 to Q4 2025 — produced a 36% BUY accuracy. The system lost to a simple buy-and-hold SPY strategy. We didn't hide this. We published it.

The culprit wasn't model complexity. It was measurement honesty: the original accuracy metric included ambiguous HOLD-zone outcomes. Once we applied minimum return thresholds and excluded HOLDs from the headline stats, the real picture emerged — and it was worse than we'd thought. That's the kind of realisation that makes a system better.

Crypto predictions remain genuinely weak: 43–55% accuracy with free daily data. That's an honest ceiling, not a temporary gap.

// What actually moved the needle

Small, compounding improvements.

Beating free-data limits required precision, not more complexity. Each gain was incremental:

+2–3%
Sector momentum features. Adding 8 cross-sector correlation and momentum features gave models market context they were missing.
+~1%
Platt scaling calibration. Raw model probabilities were overconfident. Calibration brought predicted probabilities closer to observed frequencies.
−15% false signals
Multi-horizon agreement filter. Requiring both 1-day and 5-day models to agree before signalling cut false positives significantly.
live edge
Honest accuracy measurement. The single biggest gain came from fixing the metric — not from a fancier model.
Methodology

Walk-forward validation.
The way practitioners do it.

Traditional backtests train on the full dataset and then measure performance on the same data. That's not validation — it's memorisation. Alpha Signal uses walk-forward validation exclusively: models see only what a real investor would have known at each point in time.

// 01
📊
Free Data Only
Yahoo Finance, Polygon free tier, CoinGecko, FRED macro data. 104 engineered features per asset including price, volume, volatility, momentum, macro indicators, and 8 sector momentum/correlation features.
// 02
🔄
24 Quarterly Windows
Q1 2020 through Q4 2025. Each window trains on all prior data, then tests on the next quarter with an expanding window. Models are retrained from scratch each fold. No future data ever enters the training set.

Autonomous Agent
Agent Alpha monitors live accuracy hourly. When a model's accuracy drops below 45%, it proposes and auto-executes a retrain. Regime detection scales position exposure from 1.0× (BULL) down to 0.18× (CRISIS) automatically.
The Ensemble

Six models.
One decision.

No single model maintains an edge across all market regimes. The ensemble combines classical statistics, gradient boosting, and deep learning — each casts a vote, weighted by inverse validation loss. A BUY signal requires ensemble P(up) > 0.55 and walk-forward accuracy > 51%, both the 1-day and 5-day horizon models in agreement.

// Classical · Gradient Descent
Linear Regression
Baseline return predictor. 3,000 epochs, 10 features. Captures linear factor exposures — momentum, mean-reversion, volume trends. Interpretable weights make it useful for debugging the ensemble.
Interpretable
// Classical · Sigmoid Loss
Logistic Regression
Binary UP/DOWN classifier. 3,000 epochs, regularised. Fast, stable, and provides calibrated probability estimates via Platt scaling. Acts as the ensemble's probabilistic anchor.
Probability-calibrated
// Boosting · Custom Rust CART
Gradient Boosted Trees
Custom Rust implementation — CART trees with boosting, no external library dependency. Captures non-linear feature interactions. Robust to price outliers in trending and volatile market conditions.
Non-linear
// Boosting · C++ Bindings
LightGBM
Leaf-wise tree growth with histogram binning. Faster than XGBoost at comparable accuracy. Provides a second independent boosting vote, giving the ensemble diversity in its tree-based component.
High-speed
// Deep Learning · Candle-NN
LSTM + GRU
Long Short-Term Memory and Gated Recurrent Unit networks learn sequential price patterns across history. hidden_size=64, seq_length=10–20. The only model capturing temporal dependencies and regime memory.
Temporal patterns
// Meta-Model · 5-Level Detection
Market Regime Classifier
Detects BULL / NEUTRAL / WARNING / BEAR / CRISIS from SPY returns, VIX levels, and a composite risk score. Scales position sizing from 1.0× in BULL down to 0.18× in CRISIS. Suspends SHORT signals when accuracy is below threshold.

Platform Architecture

Four layers.
One NUC in London.

Built entirely in Rust, running 24/7 on a home Intel NUC11 (Ubuntu 24.04, 64 GB RAM). Every layer connects to the next via a typed REST API. No managed cloud services in the critical path. Select a diagram below — these are sized to read without zooming.

All four layers run as systemd services on a single Intel NUC11 (Ubuntu 24.04, 64 GB RAM). REST APIs are typed and versioned. AutoHealer monitors all services and can restart any layer automatically within seconds of a failure. Open-source: github.com/Hass1966/autohealer
The agent loop runs autonomously every hour. When any model's accuracy drops below 45%, it raises a retrain proposal (48-hour cooldown prevents thrashing). Regime classification adjusts all position sizes in real time — a CRISIS detection halves maximum exposure within one cycle.

The 56% live accuracy comes from out-of-sample signals only — every prediction was made before the outcome was known. The system has been running in this configuration since March 15, 2026.

56% doesn't sound dramatic, but at scale and with disciplined position sizing, even a 6-point edge above random compounds. The +12 bps expected value per signal is the number that matters operationally — it captures both win rate and average magnitude of correct predictions.

Stocks and FX are stronger performers at 57–65% accuracy in training windows. Crypto remains the hardest domain with free daily data; the signal there is real but narrower. The system distinguishes these categories and sizes positions accordingly.

The walk-forward backtest (Q1 2020 – Q4 2025, 24 quarterly windows) validated the methodology. The old model generation showed 36% accuracy. The current generation, with sector features, Platt calibration, and the multi-horizon filter, reaches 56% live. Each improvement was small — they compound.
